= Ammitto UK Sanctions Data Repository

== Purpose

This repository stores downloaded and processed sanctions data from the
United Kingdom (UK) Office of Financial Sanctions Implementation (OFSI)
Consolidated List and related legal instruments.

The data is maintained by the Ammitto project and used by the `ammitto` Ruby
gem for querying international sanctions information.

== Legal Basis

The UK sanctions are implemented through:

* The Sanctions and Anti-Money Laundering Act 2018 (SAMLA)
* The Sanctions (EU Exit) Regulations 2019
* Various country-specific sanctions regulations

Post-Brexit, the UK maintains its own autonomous sanctions regime separate
from the EU, though many designations initially mirrored EU listings.

== Data Source

* **Authority**: Office of Financial Sanctions Implementation (OFSI), HM Treasury
* **API Endpoint**: https://ofsistorage.blob.core.windows.net/publishlive/2022-04-06/ConList.xml
* **Format**: XML
* **Update Frequency**: As changes occur in UK sanctions regimes
* **Official Source**: https://www.gov.uk/government/publications/financial-sanctions-consolidated-list-of-targets

```

│   └── ...
└── supporting/                              # Supporting data
    ├── document-types.yml                   # Document type definitions
    └── organizations.yml                    # Organization definitions
```

=== Schema Types

Each YAML file follows one of three schema types:

. **Announcement Schema** - Standard sanction announcements with `sanction_details`
. **Modification Schema** - Temporal modifications with `measure_modifications`
. **Legal Instrument Schema** - Laws and regulations with structured content

Schema files are located in `schemas/`:

* `schemas/uk-announcement.yml` - For sanction list announcements
* `schemas/uk-measure-modification.yml` - For measure modifications
* `schemas/uk-legal-instrument.yml` - For legal instruments (regulations, acts)

== Source Data Format

=== Sanction Announcement Schema

Standard sanction announcements with `sanction_details`:

[source,yaml]
----
# yaml-language-server: $schema=../../schemas/uk-announcement.yml
---
id: uk/rus0141
announcement:
  title: "Asset freeze and travel ban imposed on Rustam Temirgaliev"
  url: "https://www.gov.uk/government/publications/..."
  publish_date: "2020-12-31"
  publish_time: "00:01"                      # Optional: HH:MM format
  document_type: uk/regulation
  authority: uk/ofsi
  publisher: uk/hmt
  lang: en

regime: russia

sanction_details:
  - entity:
      type: person                            # or organization, vessel, aircraft
      name:
        full_name: "Rustam Ilmirovich Temirgaliev"
        script: Latn
        first_name: Rustam
        last_name: Temirgaliev
      aliases:
        - "Rustam Ilmyrovych Temirgaliev"
        - "Рустам Iльмирович ТЕМIРГАЛIЄВ"
      identifiers:
        - type: "unique_id"
          value: "RUS0141"
        - type: "ofsi_group_id"
          value: "12924"
      addresses:
        - country: "Ukraine"
          country_code: UA
          city: "Simferopol"

    measures:
      - type: asset_freeze
        description:
          en: "All funds and economic resources belonging to or owned by this person are frozen"
      - type: travel_ban
        description:
          en: "Travel restrictions apply to this individual"
      - type: trust_services_sanctions
        description:
          en: "Trust services sanctions apply"
      - type: director_disqualification_sanction
        description:
          en: "Director disqualification sanction imposed on 09/04/2025"

    reasons:
      - category: "territorial_integrity"
        description:
          en: |
            As former Deputy Prime Minister of Crimea, Temirgaliev played a relevant role
            in the decisions taken by the 'Supreme Council' concerning the 'referendum'
            of 16 March 2014 against the territorial integrity of Ukraine...

    period:
      effective_date: "2020-12-31"
      is_indefinite: true

legal_citations:
  - legal_instrument_id: uk/russia-sanctions-regulation-2019
    citation_type: legal_basis

group_id: uk/2020-12-31-temirgaliev
----

=== UK-Specific Measure Types

[cols="1,3"]
|===
| Measure Type | Description

| `asset_freeze`
| Financial sanctions - freeze of assets

| `travel_ban`
| Travel restrictions - entry ban

| `arms_embargo`
| Restrictions on arms trade

| `trust_services_sanctions`
| Restrictions on trust services

| `director_disqualification`
| Disqualification from director positions

|===

=== Entity Types

The UK sanctions list contains the following entity types:

Person:: Individual persons subject to sanctions
Organization:: Legal entities, companies, and organizations
Vessel:: Ships and maritime vessels subject to sanctions
Aircraft:: Aircraft subject to sanctions

== Validation and Generation

=== Rake Tasks

The repository provides rake tasks for managing the knowledge graph generation:

[source,shell]
----
# Clean all generated API files
bundle exec rake clean

# Validate source YAML files against schemas
bundle exec rake validate

# Generate API files from source YAML
bundle exec rake generate

# Validate and generate in one step
bundle exec rake build

# Clean and regenerate (recommended for updates)
bundle exec rake regenerate
----

=== Validation

The validation process checks all YAML files against their corresponding schemas:

[source,shell]
----
bundle exec rake validate
----

The validator determines schema type based on:

. File path patterns (e.g., `sanction-lists/` → announcement schema)
. Data structure (e.g., presence of `sanction_details`)

== Harmonized Data Format

After harmonization, entities are transformed into JSON-LD format using the
Ammitto gem's harmonized models.

=== Entity (JSON-LD)

[source,json]
----
{
  "@context": "https://www.ammitto.org/schema/v1/context.jsonld",
  "@id": "https://www.ammitto.org/entity/uk/rus0141",
  "@type": "PersonEntity",
  "names": [
    {
      "@type": "NameVariant",
      "fullName": "Rustam Ilmirovich Temirgaliev",
      "script": "Latn",
      "isPrimary": true
    }
  ],
  "sanctionEntries": [
    {
      "@id": "https://www.ammitto.org/entry/uk/consolidated-list/rus0141",
      "@type": "SanctionEntry",
      "authority": {
        "@id": "https://www.ammitto.org/authority/uk/ofsi",
        "name": "Office of Financial Sanctions Implementation"
      },
      "regime": {
        "@type": "SanctionRegime",
        "name": "Russia Sanctions"
      },
      "effects": [
        {
          "@type": "SanctionEffect",
          "effectType": "asset_freeze",
          "scope": "full"
        }
      ],
      "status": "active",
      "period": {
        "@type": "TemporalPeriod",
        "effectiveDate": "2020-12-31",
        "isIndefinite": true
      }
    }
  ]
}
----

=== IRI Structure

All resources use normalized IRIs:

```
# Entity (list-agnostic)
https://www.ammitto.org/entity/uk/{local_id}

# Entry (list-specific)
https://www.ammitto.org/entry/uk/{list_type}/{local_id}

# List
https://www.ammitto.org/list/uk/{list_type}

# Legal Instrument
https://www.ammitto.org/legal-instrument/uk/{instrument_id}

# Authority
https://www.ammitto.org/authority/uk/{authority_id}
```

=== Available List Types

The UK has one list type: `consolidated-list`

== Knowledge Graph Integration

This repository's source YAML data feeds into the Ammitto knowledge graph
using a normalized structure.

=== Data Flow Architecture

The harmonization process transforms source YAML data into JSON-LD knowledge graph nodes:

....
┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│   Source YAML       │     │   Ammitto Gem       │     │   JSON-LD Output    │
│   (sources/)        │────▶│   Transformer       │────▶│   (api/)            │
│                     │     │                     │     │                     │
│ • Announcement      │     │ • UkAnnouncement    │     │ • Entity nodes      │
│ • Sanction details  │     │ • UkEntity          │     │ • Entry nodes       │
│ • Legal instruments │     │ • UkMeasure         │     │ • Regime nodes      │
│ • Modifications     │     │ • Transformer       │     │ • Authority nodes   │
└─────────────────────┘     └─────────────────────┘     └─────────────────────┘
....

----
